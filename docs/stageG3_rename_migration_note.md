# Stage G3.1 Rename Migration Note

## Scope
- Rename target label column from legacy name `label_tp1_hit_h5m` to semantic name `label_tp1_hit_h20m`.
- Horizon semantics are unchanged: TP1 hit within **20 bars** on 1m anchor.

## Compatibility
- Current dataset builder writes both columns:
  - `label_tp1_hit_h20m` (primary)
  - `label_tp1_hit_h5m` (deprecated alias, same value)
- LightGBM trainer uses `label_tp1_hit_h20m` as primary and falls back to `label_tp1_hit_h5m` only if needed.

## Equivalence
- `label_tp1_hit_h20m == label_tp1_hit_h5m` for all rows generated by the updated builder.

## Deprecation Plan
1. **Now (G3.1)**: dual-write + fallback read (safe migration).
2. **After two full retrain cycles without fallback usage**: remove fallback read in trainer.
3. **Following cycle**: remove deprecated alias write from dataset builder.
